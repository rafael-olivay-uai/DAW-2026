## CRITERIO DE EVALUCIÓN

En esta Actividad Inicial se introducen los primeros comandos y conceptos necesarios para comenzar a trabajar con Git y GitHub, vinculando un repositorio local con un repositorio remoto creado desde cero.


Se evaluará que el estudiante logre:

• instalar Git correctamente como sistema de control de versiones;

• crear un repositorio remoto en GitHub;

• crear un repositorio local y vincularlo correctamente con el repositorio remoto;

• crear y versionar un archivo README.md como primer archivo del proyecto;

• crear una rama paralela a partir de la rama principal;

• realizar modificaciones sobre el archivo README.md en una rama secundaria;

• subir la nueva rama al repositorio remoto;

• generar una Pull Request hacia la rama principal;

• analizar y mergear la Pull Request desde GitHub;

• entregar en tiempo y forma el enlace solicitado.



## ENUNCIADO

La problemática de la Clase 01 consiste en crear un repositorio en GitHub, vincularlo con un repositorio local mediante Git y realizar un flujo de trabajo básico utilizando ramas y Pull Requests.


Para ello, deberán completar los siguientes pasos:

1. Crear un repositorio remoto en GitHub.

2. Crear un repositorio local en su equipo y vincularlo con el repositorio remoto.

3. Crear un archivo README.md como primer archivo del proyecto.

4. Agregar, commitear y pushear el archivo al repositorio remoto.

5. A partir de la rama principal (main), crear una rama secundaria.

6. Realizar modificaciones en el archivo README.md desde esa nueva rama.

7. Pushear la rama secundaria al repositorio remoto.

8. Crear una Pull Request en GitHub apuntando a la rama principal.

9. Revisar la Pull Request y realizar el merge correspondiente.

10. Responder a la Actividad Inicial en Classroom con el enlace del repositorio de GitHub antes de la fecha límite establecida.


## INSTRUCCIONES

Para realizar esta actividad, tengan en cuenta las siguientes indicaciones:

• El repositorio deberá estar correctamente creado y accesible desde GitHub.

• El archivo README.md deberá encontrarse visible en la rama principal del repositorio.

• La rama secundaria deberá evidenciar una modificación real sobre el README.md.

• La Pull Request deberá haber sido creada y mergeada correctamente.

• Se recomienda que el README.md incluya, como mínimo:

• nombre de la materia,

• nombre y apellido del estudiante,

• comisión o curso,

• breve descripción del objetivo del repositorio.
